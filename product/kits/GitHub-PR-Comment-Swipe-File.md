# GitHub PR Comment Swipe File

Short, clean comments for common open-source situations.

## Claiming Work

```markdown
I can take this and keep the PR focused on the acceptance criteria.
```

## Submitting

```markdown
Submitted PR #123.

Summary:
- 

Verification:
- 
```

## Asking For Review

```markdown
This is ready for review. I kept the change scoped to the issue and included verification steps in the PR description.
```

## Handling Feedback

```markdown
Thanks, updated. I addressed the requested change and re-ran:
- 
```

## Declining Scope Creep

```markdown
That makes sense as a follow-up. To keep this PR focused, I left it scoped to the original issue.
```

## Payment Without Spam

```markdown
Thanks for reviewing. If this is accepted for the bounty, please let me know the preferred payment flow.
```
